# Plan 01-02: SVG-Bilder pro BMS-Kapitel erstellen

## Objective

Hochqualitative SVG-Bilder fuer alle BMS-Kapitel erstellen. Mindestens 2 visuelle Elemente pro Unterkapitel (SVG, Diagramm oder imageUrl). Bilder muessen lernfoerdernd und visuell ansprechend sein.

## must_haves

- Mindestens 2 visuelle Elemente pro Unterkapitel
- SVG-Bilder sind korrekt, skalierbar und lernfoerdernd
- Markdown-Bilder referenzieren existierende Dateien
- npm run audit-images besteht
- npm run build fehlerfrei

## Tasks

<task id="1" title="Bestandsaufnahme visuelle Elemente">
- npm run audit-images ausfuehren
- Liste aller Unterkapitel mit weniger als 2 visuellen Elementen
- Bestandsaufnahme existierender SVGs in public/grafik-*.svg
- Bestandsaufnahme DiagramSVG.tsx Komponenten
- Priorisierung: welche Kapitel brauchen am dringendsten Bilder?
</task>

<task id="2" title="SVG-Bilder fuer Biologie erstellen" depends_on="1">
Pro Unterkapitel ohne genuegend Bilder:
- Thematisch passende SVG-Diagramme erstellen
- Beispiele: Zellaufbau, Membrantransport, Mitose-Phasen, DNA-Replikation, Stammbaum
- SVGs in public/grafik-bio-*.svg ablegen
- Markdown-Referenzen im Kapitelinhalt einfuegen: ![Alt](/grafik-bio-*.svg)
- Oder {{DIAGRAM}} Platzhalter mit diagram-Property
</task>

<task id="3" title="SVG-Bilder fuer Chemie erstellen" depends_on="1">
- Atommodell, Periodensystem-Ausschnitt, Orbitalmodelle
- Saeure-Base-Gleichgewicht, Redox-Schemata
- Organische Strukturformeln, Reaktionsmechanismen
- In public/grafik-ch-*.svg ablegen
</task>

<task id="4" title="SVG-Bilder fuer Physik erstellen" depends_on="1">
- Kraftdiagramme, Hebel, Schwingungen
- Wellenlaenge/Frequenz, Brechung, Linsen
- Schaltkreise, E-Felder, Thermodynamik-Diagramme
- In public/grafik-ph-*.svg ablegen
</task>

<task id="5" title="SVG-Bilder fuer Mathematik erstellen" depends_on="1">
- Funktionsgraphen, Koordinatensystem
- Vektoren, Dreiecke, Einheitskreis
- Wahrscheinlichkeitsbaum, Normalverteilung
- In public/grafik-ma-*.svg ablegen
</task>

<task id="6" title="Integration und Verifikation" depends_on="2,3,4,5">
- Alle SVG-Referenzen im Kapitelinhalt pruefen
- npm run audit-images (alle Unterkapitel >= 2 Bilder)
- npm run build
- Stichprobe: SVGs im Browser oeffnen und auf Qualitaet pruefen
</task>

## Verification

- [ ] npm run audit-images besteht (mind. 2 Bilder pro Unterkapitel)
- [ ] Alle SVG-Referenzen zeigen auf existierende Dateien
- [ ] npm run build fehlerfrei
- [ ] SVGs sind skalierbar und lesbar bei verschiedenen Groessen
